# سجل العمل — تطبيق الحضور والدفعات والسلفة

تطبيق ويب لتسجيل حضور الموظفين، حالة الدفع اليومية، ودفتر السلفة. يعمل بدون إنترنت بالكامل بعد أول فتح، والبيانات محفوظة فقط على الجهاز المستخدم (لا يوجد خادم أو قاعدة بيانات خارجية).

## كيف يعمل (مهم تفهمه قبل النشر)

- **لا يوجد Supabase ولا قاعدة بيانات سحابية.** كل البيانات (الموظفون، الحضور، السلفة) تُحفظ محلياً في متصفح/جهاز واحد فقط (localStorage). هذا قرار مقصود لأن الاستخدام هو شخص واحد على جهاز واحد، ويحتاج العمل بدون إنترنت — قاعدة بيانات سحابية كانت ستعقّد الأمور بلا فائدة حقيقية هنا.
- **النسخ الاحتياطي ضروري.** بما أن البيانات محلية فقط، إذا تم حذف المتصفح أو تغيير الجهاز، تُفقد البيانات. التطبيق فيه زر "تحميل نسخة احتياطية" في تبويب الإعدادات — يجب استخدامه بانتظام (مثلاً أسبوعياً).
- **Vercel هنا فقط لتوصيل الملفات،** وليس لتخزين البيانات. يمكن أيضاً فتح ملف `dist/index.html` مباشرة من أي جهاز بدون Vercel إطلاقاً، لكن النشر على Vercel يعطي رابطاً ثابتاً يسهل فتحه وتثبيته كتطبيق.

## النشر على Vercel

1. أنشئ حساباً على [vercel.com](https://vercel.com) (مجاني).
2. ادخل إلى مجلد `dist` (هذا هو الإنتاج الجاهز).
3. أسهل طريقة بدون استخدام GitHub:
   - ثبّت Vercel CLI: `npm install -g vercel`
   - من داخل مجلد `dist`، شغّل: `vercel --prod`
   - اتبع التعليمات (تسجيل دخول، اختيار اسم المشروع)
   - سيعطيك رابطاً مثل `https://اسم-المشروع.vercel.app`
4. افتح هذا الرابط من جوال أو لابتوب والدك، واطلب منه "إضافة إلى الشاشة الرئيسية" (Add to Home Screen) ليصبح كأنه تطبيق حقيقي.

## التطوير المحلي (إذا أردت تعديل الكود لاحقاً)

```bash
npm install
npm run dev      # للتجربة أثناء التطوير
npm run build    # لإنشاء نسخة الإنتاج في مجلد dist
```

## هيكل المشروع

```
src/
  App.jsx              نقطة الدخول الرئيسية وشريط التبويبات
  storage.js           طبقة حفظ/تحميل البيانات (localStorage)
  dateUtils.js          دوال التاريخ والأرقام بالعربي
  screens/
    TodayScreen.jsx      شاشة اليوم — تسجيل الحضور والدفع
    EmployeesScreen.jsx  إدارة الموظفين
    LedgerScreen.jsx     دفتر السلفة
    SettingsScreen.jsx   النسخ الاحتياطي والاستعادة
public/
  manifest.webmanifest  إعدادات التطبيق كـ PWA قابل للتثبيت
  sw.js                 Service Worker لدعم العمل بدون إنترنت
```

## إذا احتجت لاحقاً مزامنة بين جهازين

إذا تغيّر الوضع لاحقاً (مثلاً والدك وأنت تحتاجان للوصول لنفس البيانات من جهازين)، عندها فقط يصبح Supabase خياراً منطقياً. البنية الحالية تسمح بإضافة ذلك لاحقاً دون إعادة بناء التطبيق من الصفر — فقط استبدال طبقة `storage.js` بطبقة تتصل بقاعدة بيانات سحابية.
